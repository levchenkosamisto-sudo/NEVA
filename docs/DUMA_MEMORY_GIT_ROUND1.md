# ДОКУМЕНТ 1 — ПАМЯТЬ NEVA v3

Три полки SQLite: ЧТО ПРОИСХОДИЛО / ЧТО ИЗВЕСТНО / КАК ДЕЛАТЬ

Каждая запись: текст + тип + статус (АКТУАЛЬНО/ОТМЕНЕНО/УСТАРЕЛО/ОБЪЕДИНЕНО) + важность 1-5 + даты + источник + вектор

Индексатор (Церебрас→Грок→qwen):
1. Извлекает факты, ставит важность 1-5
2. Проверяет противоречия → старый ОТМЕНЕНО
3. Записывает связи в граф Kuzu
4. Векторизует через multilingual-e5-base (только когда qwen не активна)

Ночной процесс 03:00: объединение дублей (порог 0.92)

Контроль устаревания: 90 дней → ТРЕБУЕТ ПРОВЕРКИ, 180 дней → УСТАРЕЛО
Решения Директора не устаревают автоматически.

Поисковый конвейер:
1. Решения/факты важность 4-5
2. Полнотекстовый поиск
3. Граф Kuzu
4. Смысловой поиск (e5-base)
5. Извлечённое из чатов
6. Сырые чаты

Клиенты: Claude Desktop (MCP), ДУМА веб-ИИ (:9100), API ИИ (:8000/memory/search)
Лог каждого запроса: кто/запрос/уровень/факт/источник/важность/статус/время
RAM: qwen активна → e5 выгружен; e5 активен → qwen в sleep
