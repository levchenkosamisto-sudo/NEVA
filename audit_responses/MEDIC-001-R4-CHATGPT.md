1. fcntl.flock() + launchctl kickstart

Конфликта нет. Race window практически отсутствует.

Механика:

flock() привязан к файловому дескриптору процесса.

При аварийном завершении процесса ядро автоматически закрывает дескриптор и снимает lock.

Новый процесс от launchd сможет получить lock только после полного завершения старого процесса.

Но есть нюанс:

launchctl kickstart -k сначала убивает старый экземпляр, затем запускает новый.

Если старый процесс завис в системном вызове и ещё не завершился, новый экземпляр стартует, увидит занятый lock и корректно выйдет через sys.exit(0).

То есть:

старый жив → lock занят
новый стартовал → lock не получил → завершился
старый умер → launchd снова поднимет
новый получил lock → работает

Вердикт: совместимо. flock() является дополнительной защитой от двойного запуска и хорошо работает вместе с launchctl.

2. Heartbeat VS launchd KeepAlive

Здесь есть реальная архитектурная коллизия.

Текущая схема:

heartbeat > 90с
↓
status_dot делает pkill
↓
launchd запускает новый процесс
↓
новый процесс ещё не записал heartbeat
↓
status_dot снова видит heartbeat старым
↓
снова pkill

Это классическая restart loop.

Нужно добавить grace period после запуска.

Вариант:

Python
Запустить
MEDIC_START = psutil.Process(pid).create_time()

if time.time() - MEDIC_START < 120:
    return  # не проверять heartbeat

Либо ещё проще:

Python
Запустить
heartbeat_age > 90
AND
process_uptime > 120

Только тогда считать зависанием.

Рекомендую правило:

heartbeat > 90с
+
процесс живёт > 120с
=
зависание

Иначе после каждого старта появляется ложный RED.

Приоритет источников истины:

launchd отвечает за факт существования процесса.

heartbeat отвечает за факт работоспособности процесса.

Они проверяют разные вещи и не заменяют друг друга.

3. Boot-счётчик + ThrottleInterval

Не дублируют функцию полностью.

Они защищают разные уровни.

ThrottleInterval

Защита launchd:

процесс падает
↓
launchd ждёт 60с
↓
новый запуск

Он только замедляет цикл рестартов.

Не понимает причину.

Boot-счётчик

Защита приложения:

3 запуска за 5 минут
↓
объявить аварийный режим
↓
push_event(red)
↓
остановиться

Он позволяет:

поднять тревогу;

записать инцидент;

уведомить директора;

отличить случайный рестарт от системной аварии.

Что оставить?

Если выбирать один механизм:

Оставить ThrottleInterval нельзя.
Он только тормозит цикл.

Оставить только boot-счётчик тоже нельзя.
При ошибке в коде launchd будет продолжать долбить рестарты.
