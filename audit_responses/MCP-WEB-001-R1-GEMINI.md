1. Поддержка MCP в веб-версии Gemini

Прямая поддержка: Нет. Текущая веб-версия Gemini (gemini.google.com) в интерфейсе браузера не поддерживает прямое подключение к кастомным MCP-серверам (Model Context Protocol), независимо от наличия платной подписки (Gemini Advanced). Google развивает экосистему расширений (Extensions), но протокол MCP на стороне потребителя (клиента) в веб-интерфейс не интегрирован.

2. Альтернативные бесплатные способы подключения через MCP

Так как веб-интерфейс закрыт для MCP, интеграцию NEVA можно реализовать следующими бесплатными методами:

Вариант А: Использование официального API Gemini (Бесплатный уровень / Free Tier)

Суть: Ты используешь бесплатный ключ API Gemini (модели Gemini 1.5 Pro/Flash) через Google AI Studio.

Связка с MCP: Роль MCP-клиента (хоста) берет на себя локальный скрипт/код NEVA (на Python/TypeScript), который подключает локальные MCP-серверы и передает контекст в модель через API.

Лимиты: Бесплатный тариф ограничен по RPM (запросы в минуту) и TPM (токены в минуту), но полностью функционален.

Вариант Б: Сторонние бесплатные MCP-клиенты с поддержкой API Gemini

Суть: Использование готовых open-source или бесплатных сред разработки (IDE), выступающих в роли MCP-клиентов.

Инструменты: Cline (расширение для VS Code) или Cursor (в рамках бесплатного лимита запросов). В них прописывается бесплатный API-ключ Gemini и конфигурируется локальный файл mcp_config.json для вызова твоих локальных утилит.

Критика и альтернативы

Критика основного подхода (Попытка связать Web-UI и локальный MCP): Попытки автоматизировать веб-интерфейс (через Selenium/Puppeteer) для проброса MCP ненадежны, нарушают ToS и приводят к блокировкам. Веб-версия создана исключительно для человека (B2C).

Альтернатива с большими затратами, но высоким эффектом: Переход на коммерческий аккаунт Google Cloud (Vertex AI) + оркестрация через LangChain/LlamaIndex. Это дает гарантированный SLA, отсутствие жестких бесплатных лимитов API и промышленную безопасность для NEVA.

Альтернатива с меньшими затратами (Локальный контур): Отказ от Gemini в пользу полностью локальной связки: Ollama (модели Qwen2.5-Coder / Llama-3.1) + локальный MCP-клиент. Нулевые затраты на API, абсолютная приватность данных, независимость от внешнего интернета.
