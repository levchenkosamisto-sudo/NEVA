A1: Вместо абсолютного приоритета Git: факты имеют собственную версионность и ссылку на документ-источник. При конфликте: подтверждённый факт > документ > событие.

A2: Хранить обе версии факта. Добавить статус: active/superseded/disputed. Разрешение конфликта не удаляет историю.

A3: Права определяются ролями: writer_fact, writer_event, reader. Проще добавлять новых агентов без изменения логики памяти.

A4: Разрешить WebAI запись в карантинный слой facts_candidate. После подтверждения Claude или Director переносить в facts.

A5: Инкрементальная ежедневно + полный пересчёт раз в неделю. Позволяет находить накопленные ошибки индекса.

A6: Журнал незаписанных изменений. Автосброс каждые N изменений или каждые N минут. Меньше риск потери памяти при аварии.

A7: Очередь транзакций через единый memory_writer. SQLite получает одного писателя. Устраняются проблемы конкуренции и starvation.

A8: Snapshot каждый час + ежедневный архив + копия в Git LFS. Три независимые точки восстановления.

A9: Белый список разрешённых типов памяти вместо чёрного. Всё остальное блокируется по умолчанию. Безопаснее.

A10: Постоянный эталонный набор 100-200 запросов вместо 20. Отдельные категории: документы/факты/события/русский язык. Метрика статистически устойчивее.

A11: TTL зависит от типа факта: технические=30 дней, проектные решения=бессрочно, контекст сессий=7-14 дней.

КРИТИКА:
Слабые места: A7 (конкурентная запись в SQLite), A10 (выборка 20 запросов мала), A11 (единый TTL для разных типов).
Сильные решения: Git как источник документов, инкрементальная индексация, запрет неподтверждённых ИИ-фактов, резервное копирование по расписанию.
