## ЧТО ИСПРАВЛЕНО

**Замечание DeepSeek 1: STARTING → DEAD без ожидания grace period**
Принято.
Логика: если в состоянии STARTING PID исчез → немедленно DEAD (не ждать grace).
Grace period нужен только чтобы не требовать HTTP/heartbeat у только что запустившегося процесса.
Не для задержки обнаружения падения.

Новый переход:
  STARTING → DEAD : PID исчез (мгновенно, без ожидания grace period)
  STARTING → ALIVE: grace period истёк + PID есть + (HTTP OK или нет HTTP)

**Замечание DeepSeek 2: pkill по PID, не по имени**
Принято.
Используем psutil.Process(pid).terminate() → wait(5с) → .kill() если ещё жив.
Это: (1) убивает именно нужный процесс, (2) даёт шанс graceful shutdown.

```python
def kill_graceful(pid: int, timeout: int = 5):
    try:
        proc = psutil.Process(pid)
        proc.terminate()          # SIGTERM
        proc.wait(timeout=timeout)
    except psutil.TimeoutExpired:
        proc.kill()               # SIGKILL если не ответил
    except psutil.NoSuchProcess:
        pass                      # уже мёртв
```

**Замечание ChatGPT + Gemini: background_auditor без CPU критерия**
Принято.
background_auditor мониторится только по наличию PID (psutil.pid_exists).
Нет HTTP, нет log, нет CPU — только PID.
Обоснование: фоновый аудитор ожидает события и законно молчит часами.
