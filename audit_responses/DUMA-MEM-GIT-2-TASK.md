## ЗАДАЧА 1 — ЕДИНАЯ БАЗА ПАМЯТИ

### Цель
Одна база памяти и поиска для всех ИИ НЕВА.
Клод Десктоп, DeepSeek/Gemini/ChatGPT через браузер, АПИ-вызовы — все читают и пишут в одно место.

### Требования
1. Установить Mem0 локально (pip install mem0ai)
2. Создать neva_memory.py:
   - init() — создать/открыть базу neva_memory.db
   - write(content, source, tags) — записать факт/событие/документ
   - search(query, top_k=5) — гибридный поиск
   - update_index() — переиндексировать изменённые файлы
3. Добавить action memory_search и memory_write в МСП :9000 и :9100
4. Создать neva_indexer.py — индексировать governance/ docs/ state/ audit_responses/
5. Создать neva_context_pack.py — контекстный пакет перед АПИ-вызовом
6. Создать neva_session_close.py — запись итогов сессии в память

### Критерии готовности
- [ ] memory_search("ThermalGuard") находит релевантные документы
- [ ] DeepSeek через МСП :9100 пишет в ту же базу что Клод через :9000
- [ ] Индексация governance/ (50+ файлов) занимает < 60 сек
- [ ] Поиск отвечает < 500мс
- [ ] После ДУМЫ ответы аудиторов автоматически попадают в индекс
- [ ] RAM при работе Mem0 < 400 МБ

### Приоритет: P0

## ЗАДАЧА 2 — ПРОВЕРКА ПРОТИВОРЕЧИЙ ПРИ КОММИТЕ

### Цель
Ни один коммит не должен вносить противоречие в существующую базу знаний НЕВА.

### Требования
1. Создать neva_contradiction_check.py:
   - Принимает список изменённых файлов
   - Ищет похожие документы в Mem0
   - Сравнивает статусы, версии, даты, архитектурные решения
   - Возвращает: OK / WARNING [что/где] / BLOCK [критическое противоречие]
2. Зарегистрировать как pre-commit hook в .git/hooks/pre-commit
3. Типы проверок:
   - Версии компонентов (v9.4 vs v9.5)
   - Статусы задач (открыта vs закрыта)
   - Архитектурные решения (MockGraph vs реальный graphiti)
   - Даты ("утверждено 2026-06-05" vs "не утверждено")

### Критерии готовности
- [ ] Блокирует коммит при явном противоречии статуса задачи
- [ ] Предупреждает при несовпадении версий компонентов
- [ ] Не блокирует при добавлении нового несвязанного документа
- [ ] Работает за < 10 сек на типичном коммите

### Приоритет: P0

## ЗАДАЧА 3 — СТРУКТУРА ВЕТОК И АВТОКОММИТ

### Цель
Ни одно изменение не проходит мимо Гита. Автоматическая фиксация.

### Требования
1. Создать структуру веток: main / dev / audit/XXX / snap/XXX
2. Создать neva_task_start.py:
   - Принимает имя задачи
   - Создаёт ветку snap/[задача] (снапшот)
   - Создаёт ветку dev/[задача] для работы
3. Создать neva_autocommit.py:
   - Вызывается после каждой завершённой задачи
   - git add всех изменённых файлов в governance/ docs/ state/
   - git commit с форматированным сообщением
   - git push в Гитхаб
4. Добавить action git_commit и git_branch в МСП :9000
5. pre-commit hook (дополнительно к п.2):
   - Блокировать .env файлы
   - ruff check для .py файлов
   - Запретить TODO в governance/

### Критерии готовности
- [ ] git_commit("decision: принята архитектура Mem0") создаёт коммит
- [ ] snap/DUMA-MEM-GIT-2 создаётся автоматически при старте ДУМЫ
- [ ] pre-commit блокирует .env
- [ ] post-commit обновляет Mem0 индекс
- [ ] git push выполняется автоматически

### Приоритет: P1

## ЗАДАЧА 4 — ПОИСК ПО ИСТОРИИ ГИТА

### Цель
Вся история решений НЕВА доступна для поиска.

### Требования
1. Скрипт neva_git_index.py:
   - При каждом коммите берёт сообщение + список файлов
   - Записывает в Mem0 с тегами: дата, ветка, тип изменения
2. Action git_search в МСП :9000:
   - git_search(query) → [{commit, date, message, files}]
   - Ищет через Mem0-индекс коммитов
3. Нативный поиск как резерв:
   - git log --all -S "запрос"
   - Результат возвращается через action git_log

### Критерии готовности
- [ ] git_search("ThermalGuard") находит все коммиты где упоминался ThermalGuard
- [ ] git_search("задача 007") находит когда задача была закрыта
- [ ] Индекс коммитов занимает < 50 МБ

### Приоритет: P2
