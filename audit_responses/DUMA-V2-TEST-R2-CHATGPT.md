Найденные ошибки/коллизии:

Не подтверждено, что launchd использует именно окно 60 секунд во всех версиях macOS.

Не подтверждено существование фиксированного порога N для exit 78.

Не доказано, что код 78 имеет специальную обработку относительно других кодов завершения.

bootout + bootstrap может восстановить работу сервиса, но не устраняет первопричину появления EX_CONFIG.

Отсутствует экспериментальная верификация на целевой версии macOS для NEVA.

Краткое резюме изменений:

Практическая идея восстановления через bootout + bootstrap выглядит разумной.

Техническое объяснение механизма launchd содержит неподтвержденные предположения.

Для архитектурного решения NEVA требуется воспроизводимый тест.

Утверждение следует считать гипотезой, а не установленным фактом.

Критика и альтернативы:

Критика основного решения:

Решение основано на предположениях о внутреннем поведении launchd.

Отсутствуют измерения и журналы, подтверждающие модель работы.

Возможна ложная корреляция между EX_CONFIG и throttling.

Вариант с большими затратами, но более высоким эффектом:

Создать автоматический тестовый стенд launchd с сериями контролируемых завершений и собрать фактическую матрицу поведения для поддерживаемых версий macOS.

Вариант с меньшими затратами:

Использовать bootout + bootstrap как операционную процедуру восстановления.

Не полагаться на модель внутреннего счетчика launchd при проектировании ThermalGuard.
