---

КОНТЕКСТ: ПРОБЛЕМА

NEVA — персональная AI-система на Mac M1. Программы написанные Claude
часто ломаются несмотря на аудит архитектуры через 4 аудиторов (ДУМА).
Корневая причина: Claude пишет код и сам его проверяет — нет независимого
верификатора кода. Архитектура правильная, реализация нет.

---

ПРЕДЛАГАЕМОЕ РЕШЕНИЕ: SDD + TDD + ДУМА

ЭТАП 0: АУДИТ АРХИТЕКТУРЫ (ДУМА — уже работает)
  Claude → архитектура → 4 аудитора (ChatGPT/Gemini/DeepSeek/Grok)
  → 4/4 ГОТОВ → Директор утверждает

ЭТАП 1: СПЕЦИФИКАЦИЯ (Claude пишет, Директор утверждает)
  spec.md     — что делает программа (функции, входы, выходы, edge cases)
  design.md   — как устроена (классы, файлы, зависимости)
  tasks.md    — нумерованный список атомарных задач для Cursor
  tests.md    — тест-план (что проверять, ожидаемые результаты)
  AGENTS.md   — правила для Cursor (запреты, паттерны NEVA, конвенции)
  ↓ Директор утверждает spec перед кодированием

ЭТАП 2: TDD RED — тесты до кода (Claude пишет)
  tests_spec.py — все тесты написаны ДО кода, все ПАДАЮТ (нет реализации)
  Cursor запускает pytest → подтверждает что все RED
  → Это доказывает что тесты реальные, не фиктивные

ЭТАП 3: РЕАЛИЗАЦИЯ (Cursor кодирует)
  Cursor кодирует по tasks.md, одна задача за раз
  При вопросе → пишет в questions.md → Claude отвечает в answers.md
  После каждой задачи → pytest → GREEN/RED → следующая задача

ЭТАП 4: РЕВЬЮ КОДА (Claude — независимый верификатор)
  Claude читает git diff (он не писал этот код → нет слепого пятна)
  Пишет review.md: соответствие spec, edge cases, паттерны NEVA

ЭТАП 5: АУДИТ ТЕСТОВ (Claude + ДУМА)
  Claude + 4 аудитора ДУМЫ пишут финальный test_checklist.md
  Cursor выполняет каждый пункт → отчёт PASS/FAIL

ЭТАП 6: УТВЕРЖДЕНИЕ ДИРЕКТОРА
  spec ✅ + все тесты GREEN ✅ + review OK ✅ → деплой

---

КОММУНИКАЦИЯ Claude ↔ Cursor (через файлы в git репо)

  Cursor читает:  tasks.md (текущая задача)
  Cursor пишет:   questions.md (если нужно уточнение у Claude)
  Claude пишет:   answers.md (ответы на вопросы)
  Cursor пишет:   test_report.md (результаты pytest)
  Claude пишет:   review.md (ревью diff)

Никакого прямого MCP между Claude Desktop и Cursor.
Всё через файлы в git репо — просто и надёжно.
