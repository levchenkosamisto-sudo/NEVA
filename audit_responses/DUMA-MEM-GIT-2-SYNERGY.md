## БЛОК 1 — ЕДИНАЯ ПАМЯТЬ NEVA

### 1.1 Инструмент: Mem0 + multilingual-e5-base
Принято предложение Gemini: вместо e5-small фиксируем multilingual-e5-base через Ollama.
Причина: точность на русском языке ~85-90% vs ~70-75% у e5-small.
RAM: +100-150 МБ, итого ~350-500 МБ — безопасно для М1 8GB если qwen не запущен.

### 1.2 Хранение: только ссылки и эмбеддинги
Принято предложение Gemini: Mem0 хранит НЕ полный текст, а:
- Эмбеддинг вектор (ссылка на смысл)
- Путь к файлу + номера строк (file.md#L45)
- Краткое резюме (100-200 слов)
Полный текст — только в файлах Git. Экономия места в Mem0 в 3-5 раз.

### 1.3 Синхронизация записи
Принято предложение Gemini: вместо file lock → Shared Memory (Python multiprocessing.Manager).
Преимущество: нет блокировок при одновременном запросе от Клода и веб-ИИ.
SQLite WAL остаётся для транзакций, Shared Memory — для координации очереди.

### 1.4 LRU-кэш для частых запросов
Принято предложение Gemini: кэш последних 50 поисковых запросов в памяти процесса.
Попадание в кэш: ответ за < 10 мс вместо 150-300 мс.
Инвалидация кэша: при каждом memory_write.

### 1.5 Иерархия доверия с весами (уточнение)
Принято предложение Gemini: вместо жёсткой иерархии — весовые коэффициенты:
Git-файлы: 1.0 | Факты VERIFIED: 0.95 | Факты Клода: 0.8 | События веб-ИИ: 0.6 | Автофакты: 0.4
При конфликте: побеждает запись с наибольшим весом. При равных весах — уведомление Директора.

### 1.6 Три слоя памяти (финальная версия)
Слой 1 — ДОКУМЕНТЫ: .md файлы из governance/docs/state/ — только ссылки и эмбеддинги.
Слой 2 — СОБЫТИЯ: ответы ДУМЫ, решения Директора, итоги задач — полный текст (короткий).
Слой 3 — ФАКТЫ: статусы компонентов, версии, ключевые решения — с весом доверия.

### 1.7 TTL и устаревание
Принято предложение DeepSeek: вместо простого TTL — теги приоритета:
CRITICAL (вес 0.9-1.0): без TTL, только Директор может удалить.
NORMAL (вес 0.5-0.8): TTL 90 дней, метка STALE.
LOW (вес < 0.5): TTL 30 дней, автоудаление.

### 1.8 Контекстный пакет перед АПИ-вызовом
neva_context_pack.py: memory_search(тема, top_k=5) → 5 фактов в промпт.
Экономия токенов: 90% по сравнению с передачей всей истории.

## БЛОК 2 — GIT КАК АБСОЛЮТНЫЙ SSOT

### 2.1 Структура веток
main — только утверждённое Директором.
dev — текущая разработка.
audit/XXX — на время каждой ДУМЫ.
snap/XXX — снапшот перед задачей.

### 2.2 Pre-commit хук (лёгкий, < 1 сек)
Принято предложение DeepSeek + Gemini: pre-commit только быстрые проверки:
- Блок .env файлов
- ruff check для .py файлов
- TODO в governance/
Тяжёлые проверки → pre-push.

### 2.3 Pre-push хук (тяжёлый)
neva_contradiction_check.py запускается при push:
- Берёт все изменённые файлы в коммите
- Ищет похожие в Mem0 (top_k=10)
- Сравнивает версии, статусы, решения
- BLOCK при критическом противоречии
- WARNING при несущественном

### 2.4 Разрешение противоречий через Telegram
Принято предложение Gemini: при CONFLICT — Telegram-бот отправляет сообщение с кнопками:
[Принять новое] [Оставить старое] [Рассмотреть позже]
Директор нажимает одну кнопку. neva_bot.py обрабатывает callback и разрешает конфликт.

### 2.5 Бэкап базы памяти через Git
Принято предложение Gemini: sqlite3 .dump генерирует текстовый SQL-файл.
Этот файл коммитится в Git как neva_memory_dump.sql.
Преимущество: построчное версионирование, видно что именно изменилось в базе.
Дополнительно: ежечасный бинарный snапшот в GDrive.

### 2.6 Логирование обходов (--force-with-justification)
Принято предложение DeepSeek: при использовании git commit --no-verify:
Обязательно указать причину в переменной NEVA_FORCE_REASON.
Автоматически записывается в audit_responses/FORCE_COMMITS.md.

### 2.7 Поиск по истории Git
Два уровня:
Уровень 1 — git log --all -S "ключевое слово" (нативный, мгновенный).
Уровень 2 — Mem0-индекс коммит-сообщений (семантический, через memory_search).
neva_git_index.py при каждом коммите добавляет сообщение в Mem0 с тегами: дата, ветка, файлы.

### 2.8 Автокоммит (neva_autocommit.py)
Три триггера: явная команда Директора / успешные тесты задачи / закрытие Desktop.
Формат: [тип]/[задача]: [описание]

## ЧТО ПРИНЯТО ОТ АУДИТОРОВ

От DeepSeek:
- Теги приоритета CRITICAL/NORMAL/LOW вместо простого TTL
- --force-with-justification с логированием в FORCE_COMMITS.md
- Слой гиперфактов (ссылки между фактами) — отложено на P3
- Валидация базы на тестовых данных между задачами 1 и 2

От Gemini:
- multilingual-e5-base вместо e5-small
- Хранить только ссылки и эмбеддинги (не полный текст)
- Shared Memory для координации записи
- LRU-кэш для поиска
- Весовые коэффициенты доверия вместо жёсткой иерархии
- Telegram-кнопки для разрешения противоречий
- Бэкап через .dump в Git
