## БЛОК 1 — ЕДИНАЯ ПАМЯТЬ NEVA

### 1.1 Инструмент и хранилище
Mem0 (SQLite, Apache 2.0, локально на Маке). Файл neva_memory.db в ~/Documents/NEVA/.
Эмбеддер: multilingual-e5-base через Ollama (принято от Gemini — лучше e5-small для русского).
RAM: ~200-400 МБ. Не запускать одновременно с qwen (4.7GB).

### 1.2 Три слоя памяти
- Документы: полные .md файлы из governance/ docs/ state/ audit_responses/
- События: ответы аудиторов ДУМЫ, решения Директора, итоги задач
- Факты: короткие утверждения о системе (версии, статусы, архитектурные решения)

### 1.3 Иерархия доверия при конфликтах (синтез ChatGPT + архитектор)
Git-файлы > факты VERIFIED (подтверждены Директором) > подтверждённые факты (Claude) > события > автофакты WebAI

Нововведение от ChatGPT: факты имеют собственную версионность и ссылку на Git-документ источник.
При конфликте: не удалять старый, хранить оба с статусом active/superseded/disputed.

### 1.4 Права на запись (синтез всех аудиторов)
Роли вместо привязки к агентам (предложение ChatGPT):
- writer_fact: Claude Desktop, Директор
- writer_event: Claude + все WebAI через МСП :9100
- reader: все агенты через memory_search
- facts_candidate: WebAI могут писать в карантинный слой (предложение ChatGPT A4)
  После подтверждения Claude или Директора — переносить в facts.

### 1.5 Конкурентная запись (принято от ChatGPT A7)
Единый memory_writer процесс вместо прямых WAL-конкурентных записей.
SQLite получает одного писателя. Все агенты пишут через очередь к memory_writer.
Устраняет starvation и конкуренцию.

### 1.6 Индексация (синтез ChatGPT A5)
- Инкрементальная при каждом git pull (только изменённые файлы)
- Полный пересчёт раз в неделю (выявляет накопленные ошибки индекса)
- Автоиндексация ответов аудиторов ДУМЫ после каждого Круга

### 1.7 TTL и устаревшие факты (синтез ChatGPT A11 + DeepSeek)
TTL по типу факта вместо единого:
- Технические факты (версии компонентов): 30 дней
- Контекст сессий: 7-14 дней
- Проектные решения, утверждённые Директором: бессрочно
- Ответы аудиторов: 180 дней (полгода)
При истечении TTL: STALE-метка, не удаление.

### 1.8 Качество поиска (синтез ChatGPT A10)
Постоянный эталонный набор 100-200 запросов вместо 20.
Четыре категории: документы / факты / события / русский язык отдельно.
При < 75% по русскому: переключить на multilingual-e5-base.
Тест раз в неделю автоматически.

### 1.9 Резервное копирование (синтез ChatGPT A8)
Три независимые точки:
- Hourly: sqlite3 .backup → neva_memory_backup.db
- Daily: GDrive
- Weekly: Git LFS (для больших баз)

### 1.10 Синхронизация с WebAI (критика Gemini — принята)
WebAI не обращаются к neva_memory.db напрямую.
Все обращения только через МСП :9100 (neva_audit_tools.py) который проксирует к neva_memory.py.
Физический файл остаётся на Маке.

## БЛОК 2 — GIT КАК АБСОЛЮТНЫЙ SSOT

### 2.1 Структура веток
main: только утверждённое Директором
dev: текущая работа
audit/DUMA-XXX: на время ДУМЫ
snap/XXX: снапшот перед задачей (точка отката)

### 2.2 Хуки (принята критика Gemini — pre-commit лёгкий)
pre-commit (лёгкий, < 1 сек): блок .env + ruff check + TODO в governance/
pre-push (тяжёлый): neva_contradiction_check.py — проверка противоречий с Mem0
post-commit: обновить Mem0 индекс + Телеграм уведомление Директору

### 2.3 Проверка противоречий (принята рекомендация DeepSeek)
neva_contradiction_check.py в pre-push (не pre-commit — медленный).
Запускается асинхронно через neva_session_close.py в конце сессии.
Блокирует только merge dev → main при критическом противоречии.
Флаг --force-with-justification (предложение DeepSeek) обходит блокировку с обязательным логированием в FORCE_COMMITS.md.

### 2.4 Поиск по истории
Уровень 1: git log --all -S "слово" (быстрый)
Уровень 2: Mem0-индекс коммит-сообщений (семантический)
Уровень 3: neva_git_search.py — индексирует все версии файлов

### 2.5 Автокоммит
neva_autocommit.py вызывается после:
- Явной команды Директора "задача закрыта"
- Успешного прохождения всех тестов
- Конца сессии Клода (neva_session_close.py)
Формат: [тип]/[задача]: [описание]

### 2.6 Слой гиперфактов (предложение DeepSeek — принято)
В Mem0 отдельный слой: связи между фактами (supersedes, contradicts, verifies).
Строит граф зависимостей решений. Позволяет отследить цепочку изменений.
