ОТВЕТЫ АРХИТЕКТОРА НА ВОПРОСЫ КРУГА 1

Q: Транспорт файлов — как синхронизируются tasks.md / questions.md / answers.md?
A: Общая файловая система. Claude и Cursor работают в одной папке проекта
   ~/Documents/NEVA/projects/[ИМЯ]/. Никаких git push/pull для передачи файлов.
   Git используется только для истории. Claude читает файлы через NEVA MCP Tool.

Q: Триггер questions.md — как Claude узнаёт о новом вопросе?
A: Inspector мониторит папку проекта. При появлении questions.md пишет
   уведомление в claude_inbox. Claude читает при следующем обращении Директора.
   Не автоматически — Директор инициирует цикл.

Q: Кто запускает pytest?
A: Cursor запускает pytest через встроенный терминал Composer.
   Результат пишет в test_report.md. Claude только читает отчёт, не запускает.

Q: Прогресс Директора?
A: Telegram уведомления после каждого этапа + neva_control_center :9001.
   Директор утверждает переход Этап 1→2 и финальный деплой.

Q: Блокировка Cursor?
A: Cursor пишет BLOCKED: [описание] в questions.md и останавливается.
   Claude читает, отвечает в answers.md. После 3 итераций — эскалация Директору.

Q: Структура репо?
A: Монорепо NEVA. Подпапка projects/[ИМЯ]/ содержит все артефакты.
   Отдельный репо только для крупных проектов с внешними контрибьюторами.

Q: AGENTS.md в контексте Cursor?
A: Cursor Agent автоматически подхватывает AGENTS.md из корня папки проекта
   (стандарт Cursor v0.46+). Дополнительных действий не нужно.

Q: Верификация RED?
A: Claude читает полный вывод pytest из test_report.md.
   Реальный RED: ошибки ImportError / AttributeError (нет реализации).
   Фиктивный: 0 passed 0 failed, или тесты проходят без кода.

Q: Падение Claude Desktop?
A: Inspector мониторит claude_desktop. При падении — Telegram Директору.
   Все артефакты на диске — прогресс не теряется. Claude восстанавливается
   читая tasks.md + test_report.md текущего состояния.
