ФИНАЛЬНОЕ РЕШЕНИЕ АРХИТЕКТОРА (после анализа альтернатив Круга 2)

С учётом ваших альтернатив предлагаю финальную архитектуру.

ПАЙПЛАЙН — 4 ЭТАПА:

ЭТАП 1: СПЕЦИФИКАЦИЯ (Claude Desktop)
  Артефакты которые пишет Claude:
    spec.md       — требования, контракты вход/выход, edge cases
    AGENTS.md     — правила для Cursor: запреты, паттерны NEVA, стиль кода
    tests_spec.py — тесты написанные ДО кода (контракты, не реализация)
  Директор утверждает → переход к Этапу 2
  Токены Claude Desktop: ~500

ЭТАП 2: РЕАЛИЗАЦИЯ (Cursor)
  Cursor читает spec.md + AGENTS.md + tests_spec.py из папки проекта
  Cursor: пишет код → запускает ruff/mypy → запускает pytest
  Результат: test_report.md (полный вывод pytest)
  При блокировке:
    Cursor пишет questions.md (тег BLOCKED: [причина])
    fswatch обнаруживает файл → Telegram уведомление Директору
    Claude читает questions.md → пишет answers.md
    Cursor продолжает
  После 3 итераций без решения → эскалация Директору
  Токены Claude Desktop: 0 (только при блокировке ~200)

ЭТАП 3: РЕВЬЮ (Claude Desktop)
  Claude читает: git diff + test_report.md
  Проверяет:
    - каждый контракт из spec.md покрыт тестом?
    - каждый edge case проверен?
    - код соответствует AGENTS.md?
    - нет заглушек (TODO, pass, raise NotImplementedError)?
  Пишет: review.md с оценкой ГОТОВ / НЕ ГОТОВ + список замечаний
  Если НЕ ГОТОВ → Cursor исправляет → повтор Этапа 3 (до 3 циклов)
  Токены Claude Desktop: ~300-500

ЭТАП 4: УТВЕРЖДЕНИЕ (Директор)
  Директор читает review.md
  Утверждает → git tag release → деплой
  Токены Claude Desktop: 0

---

СТРУКТУРА ФАЙЛОВ:

~/Documents/NEVA/projects/[ИМЯ]/
  spec.md          ← Claude пишет на Этапе 1
  AGENTS.md        ← Claude пишет на Этапе 1
  tests_spec.py    ← Claude пишет на Этапе 1
  src/             ← Cursor пишет на Этапе 2
  tests/           ← Cursor дополняет на Этапе 2
  test_report.md   ← Cursor генерирует на Этапе 2
  questions.md     ← Cursor пишет при блокировке
  answers.md       ← Claude пишет при блокировке
  review.md        ← Claude пишет на Этапе 3

Репо: монорепо NEVA, подпапка projects/
Git: только для истории, не для транспорта файлов
