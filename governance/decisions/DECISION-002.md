# DECISION-002 — Правила индексации памяти NEVA

Дата: 2026-06-28
Статус: АКТУАЛЬНО
Директор: Серж

## Решение
Индексацию запускать с задержкой между чанками с учётом лимитов провайдеров.

## Цепочка провайдеров и лимиты
- Грок: 30 RPM → 2с между запросами
- GitHub Models: 60 RPM → 1с между запросами  
- OR GPT-OSS-120B: 20 RPM → 3с между запросами
- OR Nemotron-550B: 10 RPM → 6с между запросами
- OR Llama-70B key1: 20 RPM → 3с между запросами
- OR Llama-70B key2: 20 RPM → 3с между запросами
- Церебрас: 1 RPM → 60с между запросами
- DeepSeek: 60 RPM → 1с между запросами

## Правила
1. Не запускать индексацию сразу после тестовых запросов — ждать 60с сброса окна
2. При 429 → автоблокировка провайдера + переключение на следующего
3. min_interval = (60 / RPM) × 1.2 — запас 20% чтобы не упираться в лимит
4. Индексация большого файла (>3000 символов) разбивается на чанки по 3000 символов
5. e5-base загружается один раз на всю сессию индексации

## Отклонённые альтернативы
- Параллельная индексация несколькими провайдерами: риск суммарного превышения лимитов
- Без задержек: гарантированные 429 при файлах >3 чанков
