# РАЗДЕЛ 5: AUDIT PROTOCOL
# NEVA AUDIT PROTOCOL — РЕГЛАМЕНТ АУДИТА
Версия: 0.1-DRAFT
Статус: ПРОЕКТ — не прошёл аудит, открыт для изменений
Дата: 2026-06-01
Автор: Claude (архитектор)
Директор: не утверждён

## СТАТУС ДОКУМЕНТА
DRAFT — не является источником истины до утверждения Директором.
Изменения вносятся без процедуры двойного ревью до перевода в FIXED.

## СВОДНАЯ СХЕМА
1. ПОДГОТОВКА    -> snapshot + пакет + роли + промпты
2. КРУГ 1        -> вопросы без оценок (независимо)
3. ОТВЕТЫ        -> исполнитель отвечает + исправляет
4. КРУГ 2        -> альтернативы и оценки (независимо)
5. КОНСОЛИДАЦИЯ  -> согласие / спор -> Директор
6. ДОРАБОТКА     -> до 5 циклов -> эскалация Директору
7. ЗАКРЫТИЕ      -> чеклист + подписи + пакет -> Guardian

## ЭТАП 1 — ПОДГОТОВКА АУДИТА
Мировая практика: entry criteria (DO-178C)

Чеклист входа:
- [ ] snapshot создан в audit/snapshot_TASK-XXX/
- [ ] все тесты пройдены
- [ ] заглушки отсутствуют (проверено исполнителем)
- [ ] роли аудиторов назначены
- [ ] промпты выданы
- [ ] пакет документов передан аудиторам

## ЭТАП 2 — КРУГ 1: ВОПРОСЫ БЕЗ ОЦЕНОК
Только вопросы. Без оценок, альтернатив, критики.
Аудиторы работают независимо. Не видят вопросы друг друга.

## ЭТАП 3 — ОТВЕТЫ ИСПОЛНИТЕЛЯ
Исполнитель отвечает на ВСЕ вопросы. Пропуск не допускается.
Фиксируется в: audit/snapshot_TASK-XXX/QA_round1.md
Если выявлена проблема -> исправить + DIFF -> обновить snapshot.

Формат QA_round1.md:
  Вопрос [аудитор]-[номер]: [текст]
  Ответ исполнителя: [текст]
  Статус: [ответ принят / исправление сделано / требует обсуждения]

## ЭТАП 4 — КРУГ 2: АЛЬТЕРНАТИВЫ И ОЦЕНКИ
Каждый аудитор работает независимо по своей роли:
- Аудитор-архитектор: соответствие архитектуре НЕВА, лучшие решения
- Аудитор-антиусложнитель: упрощение, ремонтопригодность, самодиагностика
- Аудитор-ошибок-AI: галлюцинации, пропущенные требования, заглушки

## ЭТАП 5 — КОНСОЛИДАЦИЯ И СОГЛАСИЕ
Мировая практика: consensus decision making (Apache Software Foundation)
Все оценки -> audit_round2.md
Исполнитель: принимает / отклоняет с обоснованием каждое замечание.
Спорные точки -> Директор. Полное согласие обязательно.

Формат audit_round2.md:
  Замечание [аудитор]-[номер]: [текст]
  Позиция исполнителя: [принять/отклонить + обоснование]
  Ответ аудитора: [принято/не принято]
  Решение Директора: [если спор]
  Статус: [ОТКРЫТО/ЗАКРЫТО]

## ЭТАП 6 — ДОРАБОТКА (до 5 циклов)
Мировая практика: IEEE 1028, rework tracking (CMMI)

Схема:
  Замечание -> исполнитель ПРИНИМАЕТ (DIFF) или ОТКЛОНЯЕТ (обоснование)
  -> аудитор ПРИНИМАЕТ -> закрыто OK
  -> аудитор НЕ ПРИНИМАЕТ -> следующий цикл
  -> после 5 циклов -> ЭСКАЛАЦИЯ ДИРЕКТОРУ -> закрыто OK

Правила:
- Каждый цикл — новый DIFF или новое обоснование
- Повторить предыдущий аргумент без изменений — не допускается
- После закрытия замечание не переоткрывается
- Решение Директора не оспаривается

Формат rework_log.md:
  Замечание [аудитор]-[номер]: [текст]
  Цикл 1:
    Позиция исполнителя: ПРИНИМАЕТ/ОТКЛОНЯЕТ
    DIFF/Обоснование: [текст]
    Ответ аудитора: ПРИНИМАЕТ/НЕ ПРИНИМАЕТ — [причина]
    Статус: ОТКРЫТО/ЗАКРЫТО (цикл 1)
  Цикл N: (при необходимости, до 5)
    ...
  Эскалация Директору (если цикл 5 не закрыт):
    Решение Директора: ПРИНЯТО/ОТКЛОНЕНО
    Статус: ЗАКРЫТО (решение Директора)

## ЭТАП 7 — ФИНАЛЬНАЯ ПРОВЕРКА И ЗАКРЫТИЕ
Мировая практика: exit criteria (DO-178C, IEEE 1028)

Чеклист закрытия:
- [ ] все вопросы Круга 1 получили ответы
- [ ] все замечания Круга 2 закрыты или решены Директором
- [ ] rework_log.md заполнен полностью
- [ ] заглушки отсутствуют (повторная проверка)
- [ ] стандартные флаги реализованы (--health --diag --self-test)
- [ ] регрессные тесты пройдены
- [ ] аудитор ошибок AI сдал errors_log.md
- [ ] handoff checklist заполнен

Финальный пакет audit/snapshot_TASK-XXX/:
  QA_round1.md      — вопросы и ответы
  audit_round2.md   — альтернативы и решения
  rework_log.md     — все изменения по циклам
  errors_log.md     — ошибки AI
  AUDIT_CLOSED.md   — подписи + статистика

Формат AUDIT_CLOSED.md:
  Статистика аудита TASK-XXX:
    Всего замечаний: N
    Закрыто циклом 1: N
    Закрыто циклом 2-3: N
    Закрыто циклом 4-5: N
    Эскалировано Директору: N
    Снято аудитором: N
  Подписи:
    АУДИТ ЗАКРЫТ: аудитор-архитектор [дата]
    АУДИТ ЗАКРЫТ: аудитор-антиусложнитель [дата]
    АУДИТ ЗАКРЫТ: аудитор-ошибок-AI [дата]
